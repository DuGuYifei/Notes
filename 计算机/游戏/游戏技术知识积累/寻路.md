> 环境表示（Representation）→ 全局规划（Global Pathfinding）→ 路径后处理 → 局部导航/运动（Steering & Path Following）→ 局部避障（Local Avoidance）→ 群体（Crowd/Boids）→ 运动控制（Controller）。

很多概念不是互斥分类，而是跨层组合的：例如向量场可以建立在 Grid 或 NavMesh 上；Boids 本质是 steering 的群体配方；局部力避障“实现像 steering”，但语义上属于 local avoidance。

## 0. 总览：典型导航流水线（推荐心智模型）

1. **环境表示（Representation）**：Grid / NavMesh / Waypoint Graph / SDF 等
2. **全局路径规划（Global Planning）**：A* / Dijkstra / 分层 / 增量重规划等
3. **路径后处理（Post-process）**：平滑、拉直、走廊（funnel）、跟随目标点
4. **局部导航/运动决策（Local Navigation / Motion）**：Steering、Path Following
5. **局部避障（Local Avoidance）**：RVO/ORCA、势场、采样等
6. **群体（Crowd）**：Boids、队形、拥堵管理、让行规则
7. **运动控制（Controller）**：速度/加速度限制、转向半径、动画驱动/根运动

一句话：**全局给“去哪条路”，局部决定“这一帧怎么走且不撞”，控制器负责“怎么动得像”。**

## 1. 环境表示（Navigation Representation）

### 1.1 Grid（栅格/体素）

- **是什么**：把空间离散成格子（2D）或体素（3D），每格有通行/代价（cost）。
- **优点**：实现简单；便于叠加代价层；适合规则地图/RTS。
- **缺点**：分辨率与性能强绑定；路径会“走格子”，需要后处理。

常见增强：

- **多层/稀疏栅格**：分层（Hierarchical）与流式加载（streaming）
- **代价层（cost layer）**：地形、危险区、拥堵等叠加到 cost

### 1.2 NavMesh（导航网格，多边形可行走区域）

- **是什么**：用多边形覆盖可行走区域，多边形之间有邻接关系，形成图。
- **优点**：节点少，路径更自然；适合 3D 角色；走廊/拉直算法成熟。
- **缺点**：生成与动态更新更复杂；破坏式地形需要额外方案（tile 重建/局部 carve）。

关键概念：

- **最近点投射（project）**：将角色位置投射到 navmesh 上
- **poly 序列**：全局规划输出跨越的多边形序列
- **走廊（corridor）**：由 poly 序列组成的可行走通道

### 1.3 Waypoint Graph（航点图/路点图）

- **是什么**：关键点（waypoint）+ 连边（edge）的图结构（手工或程序生成）。
- **优点**：可控性强；成本低；适合道路网络、关卡式路线、空战航路。
- **缺点**：路径质量强依赖点位；细节贴边、绕障要靠局部系统补齐。

### 1.4 场（Field）：标量场/向量场/影响图

场不是“只能属于 grid”的东西，它更像是在某种表示上构建的**派生数据层**。

- **标量场（Scalar Field）**：每个位置一个数（危险度/拥堵度/通过代价）。
- **向量场（Vector Field / Flow Field）**：每个位置一个方向（推荐移动方向）。
- **影响图（Influence Map）**：常见于 RTS，把势力/火力覆盖投到地图上。

> 结论：向量场常见实现基于 Grid，但也可建立在 NavMesh 面/采样点上。

### 1.5 SDF / 距离场（Distance Field）

- **是什么**：每个位置存到最近障碍的距离（以及梯度方向）。
- **用途**：快速“离墙保持距离”、做势场避障、帮助路径平滑与贴边控制。

## 2. 全局路径规划（Global Path Planning）——“去哪”

### 2.1 基础算法（工程常用）

- **Dijkstra**：无启发式，代价非负时求最短路。
- **A\***：最常用；通过启发式 $h$ 加速搜索。

启发式要点：

- 可接受（admissible）的 $h$ 可保证最优；工程里常允许次优换速度。

更具体一点：A* 在每个节点 $n$ 上维护

- $g(n)$：从起点到 $n$ 的累计真实代价
- $h(n)$：从 $n$ 到目标的估计代价（启发式）
- $f(n)=g(n)+h(n)$：用于优先队列排序

如果 $h$ **不高估**（可接受），并且边权非负，则 A* 能保证找到最优路径。

工程常见“更快但允许次优”的变体：

- **Weighted A\***：$f(n)=g(n)+w\cdot h(n)$，$w>1$ 时更贪心、扩展更少、路径可能更长

### 2.2.1 代价（cost）到底是什么

不论 grid/navmesh/waypoint，本质都是图：点/边上有代价。

- **几何距离**：最基础
- **地形代价**：泥地更慢、上坡更累
- **风险/偏好**：远离火力覆盖区、远离噪声区

工程上常把代价写成：

$$c(n,n') = \text{distance}(n,n') \cdot \text{terrainMultiplier} + \text{extraPenalty}$$

### 2.2 Grid 常见变体/加速

- **JPS（Jump Point Search）**：均匀栅格上显著减少扩展节点。
- **Any-angle（Theta* 等）**：允许视线直连，路径更直更自然。
- **HPA\***：分层聚类，适合超大地图。

### 2.3 NavMesh 常见做法

- 在 poly 邻接图上跑 A*（节点少）。
- 输出 poly 序列后，通过走廊算法（如 funnel）得到可行走的“拉直路径”。

更具体一点：

- **搜索图的节点**：通常是 navmesh 的多边形（poly）
- **边的含义**：两个 poly 共享边界（portal），可从一个穿到另一个
- **边代价**：可用 poly 中心距离/portal 距离/或结合地形代价

全局结果常不是“可直接走的点列”，而是：

- 一串 poly（走廊）
- 走廊的 portal 序列（每个 portal 是一段线段）

接下来用 Funnel 把 portal 序列转成更短、更直的拐点序列。

### 2.4 动态与重规划（Replanning）

工程上通常用“触发策略”避免频繁抖动：

- **阈值触发**：偏离走廊/目标移动过远/路径被阻断
- **冷却时间**：重算后至少 N 秒不再重算
- **局部重算**：只重算受影响区域或短前方片段

更学术/专用：D* Lite / LPA*（适合地图持续变化且需要增量更新）。

## 3. 路径后处理（Post-processing）——“让路径可走、好看、稳定”

常见处理：

- **平滑**：消除锯齿（grid 更常见）
- **拉直**：视线可达就合并中间点（string pulling）
- **走廊拉直（Funnel）**：navmesh 的核心后处理
- **取前瞄点（look-ahead）**：给局部 steering 一个短期目标

### 3.1 Funnel（String Pulling）为什么能“拉直”

直观理解：navmesh 全局给了“你必须走在这个走廊里”，Funnel 在不离开走廊的前提下尽量走直线、减少拐点。

更具体一点：

- poly 序列会导出一串 **portal（门）**，每个 portal 是一段线段（走廊左右边界的一部分）
- Funnel 维护一个“漏斗”：
	- apex：当前已确定的拐点
	- left/right：从 apex 出发的左右边界射线
- 当新的 portal 进入时，会尝试收紧 left/right
	- 如果 left/right 不交叉：说明仍然可以不拐弯继续前进
	- 一旦发生交叉：说明必须在某个边界点拐一次，于是把 apex 固定为一个新的拐点并重置漏斗

所以 Funnel 的输出路径通常：点更少、转向更自然、严格落在走廊内。

## 4. 向量场（Flow Field / Vector Field Navigation）

### 4.1 Flow Field 是什么

对某个目标（或少数目标）计算一个“指路场”：**每个位置给出下一步推荐方向**，单位只要查表即可移动。

以 Grid 常见实现为例：

1. **Integration Field**：从目标做 Dijkstra 得到“到目标的累计代价”。
2. **Flow Field**：每格选择邻居中累计代价下降最快的方向（近似梯度）。

更具体一点（核心机制）：

- 设每个格子 $x$ 有“通过代价” $cost(x)$（可以叠加地形/危险/拥堵）
- Integration field 计算 $I(x)$：从目标出发，求到每个格子的最小累计代价（相当于在网格图上跑一次 Dijkstra）
- Flow field 给方向：对每个格子 $x$，在邻域 $N(x)$ 中选 $I$ 最小的 $y$，推荐方向 $dir(x)=normalize(y-x)$

这种做法的关键优势是：**很多单位共享目标时，只需要为“目标”算一次 $I/dir$，单位每帧查表即可**。

### 4.2 适用场景

- RTS 大量单位共享目标：一次算场，多人复用。
- 需要“自然绕开高代价区域”（危险区/泥地/拥堵）。

### 4.3 与 Grid/NavMesh 的关系（对应你的疑问）

- **常见：Grid + Flow Field**（最直观、实现成本低）。
- **也可：NavMesh + Flow**（在 poly/采样点上估计梯度方向）。

## 5. 局部导航与 Steering（Local Navigation / Steering）——“怎么走得像”

Steering 更像“运动意图”的合成器：输出期望速度/加速度/转向。

你提到的疑问（steering 是不是力/速度叠加？avoidance/boids 不也是？）可以这样理解：

- **从实现形态**：它们很多都能写成“多个向量项的加权组合”。
- **从工程分层语义**：steering 解决“想往哪走”（软意图），avoidance 更偏“不能撞”（硬约束/强目标），boids 提供“群体感/队形感”（群体层意图）。

### 5.1 常见 Steering 行为（可叠加混合）

- **Seek / Flee**：朝向/远离目标
- **Arrive**：接近目标时减速
- **Pursuit / Evade**：预测目标未来位置
- **Wander**：随机游走
- **Follow Path**：路径跟随（用前瞄点驱动）

### 5.1.1 Steering 的“叠加”长什么样

常见有两种输出形式：

- **期望速度（preferred velocity）**：
	$$v_{pref} = \sum_i w_i\, v_i$$
- **期望加速度/力（acceleration/force）**：
	$$a_{des} = \sum_i w_i\, a_i$$

然后做约束（非常关键）：

- 限速：$\|v\| \le v_{max}$
- 限加速度：$\|a\| \le a_{max}$
- 限角速度/转向半径（车辆/角色转身常用）

### 5.2 Path Following（路径跟随）的工程要点

- 不要死跟每个路径点：用 **look-ahead** 生成局部目标点
- 根据速度与转向半径做约束，避免“急拐 + 抖动”
- 与避障结合：避障推开后要有“回归走廊/回归路径”的机制

## 6. 局部避障（Local Avoidance）——“怎么不撞”

这层经常被误当作 steering 的一部分。工程上建议单列，因为避障通常是“硬约束或强目标”。

参考视频：

- [bilibili《群体寻路与避障》Steering Behavior转向行为如何实现避障智能体](https://www.bilibili.com/video/BV1yCqfYuEh1)

### 6.1 方法分类

#### A) 速度空间方法（Velocity Obstacles）

- **RVO / ORCA**：多智能体常用；把“会碰撞的速度”变成禁区，在速度空间里选安全速度。
- 更具体一点（ORCA 的工程语义）：
	- 先有一个来自 steering/path following 的 $v_{pref}$（“我想这么走”）
	- 避障把每个邻居/障碍转换成速度空间约束（常近似成半平面约束）
	- 在所有约束的交集里选一个速度 $v$，使它**尽量接近** $v_{pref}$（可理解为一个小型优化/约束求解）
- 优点：多人对称避让效果好，crowd 表现稳定。
- 难点：狭窄通道死锁、抖动、让行规则与边界条件。

#### B) 力/势场方法（Force / Potential Field）

- 对障碍产生排斥力，对目标产生吸引力，合力输出为移动意图。
- 这种方法“长得很像 steering”：
	$$a = w_g a_{goal} + \sum_j w_j a_{repulse}(obstacle_j)$$
- 优点：简单、连续平滑。
- 缺点：**局部最小值陷阱**，复杂障碍容易卡住。

> 你问“local avoidance 用局部力是否属于 steering？”
> - **实现方式**：可以写成 steering 的一项“力”。
> - **分类语义**：它的目标是“避免碰撞”，通常归为 Local Avoidance。

#### C) 采样/预测方法（Sampling-based）

- 采样候选速度/转向，短时预测轨迹，选择最优（安全 + 接近目标）。
- 常用于车辆/有最小转弯半径约束的单位（类似 DWA 的工程思路）。

### 6.2 常见工程坑

- **死锁**：狭窄通道双方互让不动，需要优先级/随机扰动/回退策略。
- **抖动**：每帧速度大幅改变，需要滤波、惩罚频繁换向、保持时间窗。
- **偏离路径**：被推离走廊太远，需要走廊约束或定期回归。

## 7. Boids（群体行为 / Flocking）

### 7.1 Boids 属于什么？（回答你的混乱点）

Boids 本质是一组 **Steering 行为的组合**，因此“概念上属于 steering”。

经典三项：

- **Separation（分离）**：别挤在一起
- **Alignment（对齐）**：朝相近方向
- **Cohesion（凝聚）**：保持群体

把它写成和 steering 一样的形式会更直观：

$$a = w_s a_{separation} + w_a a_{alignment} + w_c a_{cohesion}$$

工程上也常把 Boids 单列为 **Crowd/Group**，因为它引入：邻域查询、密度控制、编队、让行等系统问题。

### 7.2 Boids 与避障

- separation 能减少相互穿插，但**不能替代严格避障**。
- 常见组合：Boids（群体感） + ORCA（硬避障） + Path Following（目标约束）。

## 8. 推荐的组合方案（按游戏类型落地）

### 8.1 单个 NPC（动作/射击常见）

- 表示：NavMesh
- 全局：poly 图 A* + 走廊
- 后处理：Funnel + look-ahead
- 局部：steering（Arrive / Follow Path）
- 避障：ORCA/RVO 或采样避障
- 控制：速度/加速度/转向约束 + 动画驱动

### 8.2 RTS 大规模单位

- 表示：Grid（叠加 cost layer）
- 全局：Flow Field（共享目标）或 HPA*
- 局部：简单 steering + 队形/拥堵处理
- 避障：简化 ORCA 或“拥堵代价回写”到 cost

### 8.3 车辆/转向半径受限单位

- 表示：道路图或 NavMesh
- 全局：道路图 A* / 走廊
- 局部：曲率受限路径跟随（前瞄点 + 转向约束）
- 避障：采样预测更常见

## 9. 术语对照（把你的词放回正确层级）

- **grid / navmesh / waypoint graph**：环境表示（Representation）
- **pathfinding（A* 等）**：全局规划（Global Planning）
- **向量场 / flow field**：引导数据（可建立在 grid 或 navmesh 上）
- **steering**：局部运动意图合成（Local Navigation / Motion）
- **local avoidance**：局部避障（约束/碰撞避免）
- **Boids**：群体 steering 配方（也可归到 crowd）
